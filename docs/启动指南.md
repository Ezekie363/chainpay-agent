## 前置准备

### 1. Python 环境

```bash
cd hackathon/chainquery-agent
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. 环境变量

```bash
cp .env.example .env
# 编辑 .env，填写以下内容：
```

| 变量 | 说明 | 获取方式 |
|------|------|----------|
| `DEEPSEEK_API_KEY` | DeepSeek API 密钥 | platform.deepseek.com |
| `AGENT_WALLET_API_KEY` | CAW API 密钥 | Cobo Agentic Wallet 控制台 |
| `AGENT_WALLET_WALLET_ID` | 钱包 UUID | CAW 控制台 → 钱包详情 |
| `AGENT_WALLET_ADDRESS` | 钱包 EVM 地址 | CAW 控制台 → 钱包详情 |
| `SERVER_PRIVATE_KEY` | 服务端收款私钥（**仅测试**） | 自行生成，见下方说明 |
| `SERVER_WALLET_ADDRESS` | 对应地址 | 由私钥推导 |

**生成测试用服务端钱包：**

```python
from eth_account import Account
acct = Account.create()
print("私钥:", acct.key.hex())
print("地址:", acct.address)
```

> ⚠️ `SERVER_PRIVATE_KEY` 仅用于接收测试 USDC，切勿用于主网，切勿提交到 Git。

### 3. Cobo Agentic Wallet Pact 配置

Agent 钱包必须有一个**状态为 active** 的 Pact，且包含以下策略：

- **类型**：`message_sign`
- **verifyingContract**：`0x036CbD53842c5426634e7929541eC2318f3dCF7e`（Base Sepolia USDC）
- **chainId**：`84532`（Base Sepolia）

在 CAW App 中审批 Pact 后，状态变为 `active` 才能签名。

### 4. Agent 钱包充值 USDC

向 `AGENT_WALLET_ADDRESS` 转入至少 **1 USDC**（Base Sepolia 测试网）。

获取测试 USDC：[Circle USDC Faucet](https://faucet.circle.com) → 选择 Base Sepolia → 输入地址

> ⚠️ 直接转入地址即可，无需通过 Cobo 渠道。CAW 内部余额显示为 0 属正常现象，x402 Facilitator 直接读取链上余额。

## 注意事项

### MPC 签名耗时长

每次支付需要 Cobo MPC 多方签名，约 **15–25 秒**。这是正常现象，终端会显示 `Agent thinking...` 等待即可。

### 支付偶发 402 Rejection

x402 Facilitator 偶尔会拒绝第一次支付授权（返回 402），此时：
- Agent 自动静默重试一次（等待 1.5s）
- UI 只显示最终结果，不暴露中间失败
- 极少数情况下 x402 库本身会进行 recovery 签名（导致实际扣款 > 查询次数）

### 不要并发多个查询

CAW 不支持同一钱包的并发签名请求。Agent 已强制顺序执行并在每次签名间隔 0.5s，请勿手动并发调用工具接口。

### macOS 系统代理

如果 httpx 报 503 或连接被拒，可能是系统代理拦截了 localhost 流量。代码已使用 `trust_env=False` 绕过，若仍有问题请在系统设置中暂时关闭代理。

### Pact 过期

CAW Pact 有有效期（默认 24 小时）。若签名报 `MESSAGE_SIGN_DENIED`，需在 CAW App 中重新审批新的 Pact。检查当前 Pact 状态：

```python
# 快速检查
from cobo_agentic_wallet.client import WalletAPIClient
import asyncio, os
from dotenv import load_dotenv
load_dotenv()

async def check():
    async with WalletAPIClient(base_url=os.environ["AGENT_WALLET_API_URL"],
                               api_key=os.environ["AGENT_WALLET_API_KEY"]) as c:
        pacts = await c.list_pacts(wallet_uuid=os.environ["AGENT_WALLET_WALLET_ID"])
        for p in (pacts if isinstance(pacts, list) else pacts.get("data", [])):
            print(p.get("pact_id"), p.get("status"))

asyncio.run(check())
```

## 常见错误

| 错误 | 原因 | 解决 |
|------|------|------|
| `MESSAGE_SIGN_DENIED (403)` | Pact 无 `message_sign` 策略或已过期 | 在 CAW App 中创建并审批新 Pact |
| `ModuleNotFoundError: No module named 'agent'` | 未设置 PYTHONPATH | 使用 `PYTHONPATH=. python ...` 启动 |
| `CAW message_sign 超时（120s）` | MPC 节点无响应 | 检查网络，重试 |
| `Failed to handle payment: Client error '402'` | Facilitator 拒绝支付 | Agent 会自动重试，通常自愈 |
| `address already in use :8000` | 端口被占用 | `pkill -f "uvicorn server.main"` 后重启 |
| `trust_env` 相关 503 | 系统代理拦截 localhost | 代码已内置 `trust_env=False`，若仍报错关闭系统代理 |
| USDC 余额足够但支付失败 | CAW 内部余额未同步（已知限制） | 正常，代码使用 `message_sign` 绕过，以链上余额为准 |
